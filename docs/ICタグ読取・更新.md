# ICタグ読取・更新 機能仕様書（完成形）

## 概要

ICタグ（RFID）を読み取り、一覧表示したうえでステータスを変更し、DB の [ICタグ台帳] と [tag_table4] を更新する機能。メイン画面の「ICタグ読取・更新」ボタンから遷移する。

---

## 画面構成

- **タイトル:** ICタグ読取・更新
- **読取コントロール**
  - 「読取開始」/「読取停止」ボタン（実機タグリーダー連携用。現状はトグルのみ）
  - 状態表示: 「読取中…」/「停止中」
  - **テスト用:** 「テスト: ランダムに1件読み取り」ボタン（API からランダム1件を取得し、読み取りとして一覧に追加）
- **送信待ちバー**
  - 「送信待ち: N 件」（送信候補の件数）
  - 「送信」ボタン（送信候補を一括送信）
- **タグ一覧**
  - 背景: 白。各項目間は区切り線（Divider）。
  - 送信選択した行: 薄い緑背景＋左に青いボーダー。

---

## 一覧の1件あたりの表示内容

| 表示項目 | 説明 |
|----------|------|
| EPC | tag_id2（タグID）。モノスペース・小さいグレー文字 |
| 商品コード・番号 | あれば「商品コード: xxx  番号: xxx」。その行の右端にステータス |
| ステータス | 右端にバッジ表示（ＯＫ／修理中／貸出中／廃棄 など）。色分けあり |
| 商品名 | 太字・メイン表示 |
| 送信済み | 送信済みの行には「送信済み」を緑で表示 |

---

## 操作

| 操作 | 動作 |
|------|------|
| **ダブルタップ** | 送信候補の追加/解除（薄い緑・左ボーダーで選択状態を示す） |
| **長押し または 右クリック** | ステータス選択ダイアログを表示。選択したステータスに更新し、**その行を送信候補に自動追加** |
| **送信ボタン** | 送信候補の行を順に API に送信。[ICタグ台帳] と [tag_table4] を更新。成功した行は「送信済み」表示にし、送信候補から外す |

---

## ステータス候補

ＯＫ、在庫、自社使用、修理中、整備中、清掃中、貸出中、登録、廃棄、予約、不明（tag_mode2 の想定値に合わせている）。

---

## テスト機能

- **「テスト: ランダムに1件読み取り」**
  - タグリーダーなしで動作確認するためのシミュレート。
  - **GET** `/api/products/random?count=1` で [ICタグ台帳]＋[商品台帳] からランダム1件を取得し、読み取りとして一覧に1行追加する。
  - 追加した行もダブルタップ・ステータス変更・送信が可能。

---

## バックエンド連携（API）

### 送信時（POST /api/product-updates）

- **リクエスト:** `epc`, `read_at`, `changes.status`, `user_id`（担当者コード） など。
- **処理内容:**
  1. **[ICタグ台帳]** を更新  
     - 条件: `LTRIM(RTRIM([tag_id2])) = epc`  
     - 更新列: `[tag_mode2]` = 送信したステータス、`[updated]` = サーバー現在時刻（**秒単位**、コンマ秒なし）
  2. **[tag_table4]** にログ1件を INSERT  
     - `read_time`: サーバー現在時刻（秒単位）  
     - `tag_id2`: epc  
     - `tag_mode`: 送信したステータス  
     - `s_code`: user_id を整数にした値（従業員コード）。無効なら NULL  
     - `受付番号`: 0  
     - `mark`: 0  
     - `complete`: 1  
     - `Remarks`: NULL  

### 参照・テスト用

- **GET** `/api/products?epc=...` … EPC で商品取得（[ICタグ台帳]＋[商品台帳]）
- **GET** `/api/products/random?count=1` … ランダム1件取得（テスト用）

---

## DB テーブル（参照）

- **[ICタグ台帳]** … 更新対象。`tag_id2`, `tag_mode2`, `updated` 等。
- **[tag_table4]** … タグモード更新ログ。`tag_id2` ごとの履歴として利用。
- **[商品台帳]** … 商品名取得用（JOIN）。
